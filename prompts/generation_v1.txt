Generate exactly [N] LinkedIn posts.

Your job is to generate sharp, high-signal B2B LinkedIn posts that feel specific, credible, and native to the platform.
Do not produce generic, vague, repetitive, or advice-blog-style content.

POST REQUIREMENTS
For each post:

1. Hook archetype
- Choose hook_archetype from the style guide hook archetypes.
- If none fits perfectly, use the nearest existing archetype rather than inventing a new category.

2. Credibility
- Include at least one credibility signal in the post body.
- Valid credibility signals include:
  - a specific number
  - a named tool
  - a dated event
  - a concrete workflow step
- Do not invent statistics, URLs, trend facts, or experiences not grounded in the provided inputs.

3. Length and format
- Body length must be between [MIN_CHARS] and [MAX_CHARS] characters.
- The first line must be 120 characters or fewer.
- Use plain text only.
- Use LinkedIn-native line breaks.
- Do not use markdown bullets, headings, or code fences.

4. Post quality
- Each post should feel distinct in angle and framing.
- Avoid filler, vague motivation language, and generic takeaways.
- Prefer concrete workflows, tradeoffs, observations, inefficiencies, mistakes, or surprising insights.
- The body should sound like practitioner-level thinking, not templated content marketing.
- Keep cadence conversational: mix short and medium/long sentences for natural rhythm.
- One optional sharp opinion or light joke/sarcastic line is allowed when it fits the point.
- Concrete is required, but delivery should feel human, not robotic or over-formal.
- Do not use over-formal theatrics like "This is not hypothetical" / "This is not theory".
- Do not default to rigid numbered consultant-style lists unless the user explicitly asks for a list format.
- Do not use fake citation-style phrasing like "according to the data", "research shows", "studies prove" without a provided source.
- Avoid repetitive transition/conclusion templates (e.g., "That said,", "In conclusion,", "The takeaway is", "What do you think?" at the end of every post).

5. Trend usage
- Use trend information only from the provided Trend Brief JSON.
- If a post uses a trend, set:
  - post_type = "trend_reaction"
  - trend_source = exact source URL from the Trend Brief
- If no trend is used, set trend_source = "none".
- Never cite or imply an external trend that is not present in the Trend Brief.

6. Banned openers
Do not start the first line with:
- "In today's world"
- "Excited to share"
- "Hot take:"
- "Game changer"
- "Let that sink in"
- "I'm humbled"
- "Im humbled"

7. Contrarian exception
- You may start with "Unpopular opinion:" only when post_type is "contrarian"
- If used, the body must genuinely challenge a mainstream assumption

HOOK SCORING
For each post, internally score the hook_clarity_score from 1 to 10 using this rubric:

- Specificity (0-4)
  - 4 = contains a number, name, tool, or date
  - 3 = concrete claim
  - 2 = moderately concrete
  - 1 = vague
  - 0 = generic

- Curiosity gap (0-3)
  - 3 = strong open loop
  - 2 = mildly intriguing
  - 1 = predictable
  - 0 = flat

- Clarity / banned phrasing (0-3)
  - 3 = clear and no banned phrasing
  - 2 = one borderline phrase
  - 0 = banned phrasing present

If total score is less than 7, rewrite the hook before finalizing the post.
